# Política de Segurança

## Reportando Vulnerabilidades

Abra uma issue privada (Security Advisory) ou envie e-mail para security@example.com descrevendo:

- Versão afetada
- Passos para reproduzir
- Impacto potencial

## Processo

1. Triage inicial em até 72h.
2. Correção em branch dedicada.
3. Release de patch (ex.: v0.x.y) e aviso público.

## Ferramentas Ativas

- Dependabot para atualizações de dependências.
- CodeQL para análise estática de segurança.
- Secret scanning (ativado no repositório).
